# 🌊 今回のEGHで狙うべきブルーオーシャン（統合版）

## 1) コア戦略（結論から）

* **AIは“代替”ではなく“伴走者”**
  学習ログを見て「今日の1分」だけを出す、**提案＋励まし役**としてのAI。教師/メンターの負担を減らし、学習者の継続を後押しする。
* **初回5タップ以内で成果が出る軽量UX**
  すぐ動く・すぐ分かる・すぐ嬉しい＝**30秒で“できた！”が見える**。
* **点数以外（非認知）の可視化**
  「挑戦回数・継続日数・助け合い」を**バッジ/グラフ**で見える化。
* **“4つの断絶”に直撃**
  動機/接続/認識/応用の断絶を、**小さな成功体験×協働要素×即時可視化**で順に潰す。

> 一言サマリ：
> **“今日の1分”をAIが伴走してくれる。すぐ成果が見え、仲間とも繋がり、点数に出ない成長まで貯金されていく。**

# ✅ 積極的に選ぶべき要件（実装に落とす）

## A. 体験要件（UX）

1. **初回5タップ以内で開始**

   * サインアップ省略（ゲストOK）→「はじめる」→「今日の1分」表示→解く→その場で結果。
2. **30秒で可視化**

   * 解答後、**進捗バー＋“今日の一歩”バッジ**を即表示。
   * 「次の1分」ボタンで**摩擦ゼロの周回**。
3. **“協働の最小単位”を用意**

   * 「1分ボイス要約→AIが1文に整形→仲間に共有→“イイネ/私も挑戦”」の**1タップ循環**。
4. **“応用の断絶”に橋をかける**

   * スキル別**ミニ・実践タスク**（例：英語—買い物1分ロールプレイ、資料作成—図表1枚の清書）。

## B. データ・指標要件（見える化）

1. **学力系**：正答率、理解度（単元別）、“連続日数”
2. **非認知系**：

   * **継続力**（連続日数/週内達成数）
   * **協調性**（「助けた/助けられた」回数、リアクション数）
   * **挑戦性**（再挑戦数、新分野着手数）
3. **ダッシュボード**：

   * 週間「できる化」グラフ（先週比↑↓）
   * バッジ棚（継続力★/協調性★/挑戦性★）

## C. AI要件（“AIだからできる”箇所を限定）

1. **今日の1分レコメンド**：
   学習ログ×間違いパターン→次の1問/1分タスクを**自動提案**。
2. **1分ボイス→要約**：
   口頭アウトプット→**1行要約＋励まし**を即生成。
3. **弱点兆候の検出**：
   正誤履歴・時間超過・中断傾向から\*\*「つまずきサイン」\*\*を可視化（人では追い切れない粒度）。

> 重要：AIは\*\*“核心3点”**（提案・要約・兆候検出）に限定し、
> 「教材全部作る」「重い自動採点をフル実装」など**時間が溶ける要素は切る\*\*。

# 🧩 予選MVPの完成像（そのままIssue化OK）

## 1. ストーリーボード（7ステップ）

1. トップ：**「はじめる」**（ゲストOK）
2. 画面1：**今日の1分**（単一カード／開始）
3. 実行：**1問 or 1分タスク**
4. 結果：**即時フィードバック＋進捗バー**
5. バッジ：**“今日の一歩”獲得**（非認知メダルも更新）
6. 協働：**1分ボイス→AI要約→共有**（任意）
7. ダッシュボード：**今週の伸び・継続日数・非認知バッジ**

## 2. 画面構成（最低3画面）

* **ホーム**：今日の1分 / 継続日数 / 次の一歩
* **実行**：問題orタスク / タイマー / スキップ
* **ダッシュボード**：進捗グラフ / 非認知バッジ / 先週比

## 3. API 叩き方（非エンジニア向け手順）

* **フロント（Vercel）**

  * `GET /today` → 今日の1分カード（タイトル/所要時間/開始ボタン）
  * `POST /submit` → 回答 or 実施結果（正誤/所要時間を返す）
  * `POST /voice` → 音声→テキスト→要約（文字列）
  * `GET /dashboard` → 進捗・バッジ・先週比
* **バック（軽量API, 例：Koyebや他PaaS）**

  * データは**最小**：`user_id(ゲスト) / date / unit / result / time / streak`
  * AI呼び出しは\*\*/recommend\*\*, **/summarize**, **/detect\_flags** の3本のみ。

## 4. データスキーマ（最小・ExcelでもOK）

* `users`: id, created\_at, last\_active
* `sessions`: user\_id, started\_at, unit\_id, duration\_sec, result(enum), retry(bool)
* `stats_daily`: user\_id, date, correct\_rate, streak, coop\_points, grit\_points, challenge\_points
* `badges`: user\_id, type(継続/協力/挑戦), level(★1〜★5), awarded\_at

## 5. KPI（予選で掲げる数字例）

* **初回体験KPI**：起動→結果表示**30秒以内 達成率90%**
* **継続KPI**：**翌日継続30%**（MVP指標として十分）
* **非認知KPI**：**継続力★1獲得率40%**、**協力行動1回/人**
* **デモ信頼性**：**デモURL稼働率≈100%＋90秒動画バックアップ**

## 総合戦略：AIによる「パーソナル・ラーニング・レジャー（個人学習台帳）」の構築

我々が目指すのは、競合がひしめくレッドオーシャンである「何でもできる総合学習プラットフォーム」ではありません。我々が狙うべきブルーオーシャンは、**学習者のあらゆる努力の「プロセス」を価値ある資産に変え、AIによって個人の「スキル資本」として可視化・資本化する『パーソナル・ラーニング・レジャー』**です。

[cite_start]これは、学習コンテンツを提供するのではなく、**「学習体験そのものに深く介入する、高付加価値AIモジュール」**として機能します。この戦略は、ハッカソンのテーマ「テクノロジーで、『学び』の体験を変える」 [cite: 3, 126] [cite_start]に対する最も鋭い解答であり、審査員の全評価項目 [cite: 5, 144] に応えるものです。

### 1. プロダクトコンセプト：なぜこれが「勝てる」のか？

[cite_start]このプロダクトは、学習者が抱える最も根深い課題である**「4つの断絶」** [cite: 4, 15]、特に**「動機の断絶」**と**「認識の断絶」**を解消することに特化します。

#### ✅ **狙うべき3つのブルーオーシャン領域**

1.  **学習者の「非認知能力」を“見える化”し、育む領域【創造性・将来性】**
    * [cite_start]**コンセプト**: AIがGitHubの活動履歴などの学習プロセスを分析し、「粘り強さ」「協調性」といった、従来のテストでは測れない**非認知能力**を客観的なスコアやバッジとして可視化します [cite: 7, 13]。これは、既存サービスが手をつけていない、極めて独創的なアプローチです。
    * **価値**: 教師や本人も気づきにくい内面的な成長をデータで示し、より深い自己理解と動機付けに繋げます。

2.  **教師・メンターを「スーパーコーチ」に変える支援領域【実用性・将来性】**
    * [cite_start]**コンセプト**: atama+の成功事例 [cite: 7, 22] に倣い、教師を代替するのではなく「支援」します。生徒の学習データをAIがリアルタイムで分析し、「クラスの30%がこの単元で躓いています」といったアラートを教師用ダッシュボードに通知します。
    * **価値**: 教師を画一的な進捗確認作業から解放し、個別の声かけや励ましといった、人間にしかできない高付加価値なコーチング業務に集中させます。

3.  **社会人の「リスキリング」における“最初の1歩”と“継続”の支援領域【実用性・市場性】**
    * [cite_start]**コンセプト**: 「何を学べばいいかわからない」「忙しくて続かない」という社会人の課題に対し、AIがキャリアパスを診断し、**「通勤中の5分で達成できるマイクロタスク」**を毎日提案します [cite: 15]。
    * [cite_start]**価値**: 「5分で1勝」という小さな成功体験を高速で積み重ねる**「低摩擦ループ」** [cite: 4, 9] を提供し、学習の習慣化を強力にサポートします。

### 2. プロダクトの具体的要件

上記のコンセプトを、審査員が触って感動するレベルの「動く実物」に落とし込みます。

#### ✅ **機能要件**

* [cite_start]**超個別最適化マイクロラーニング**: AIがユーザーのGitHubリポジトリや学習ログを分析し、「今日の弱点克服1分チャレンジ」のような超具体的なタスクを自動生成します [cite: 12, 13, 15]。
* [cite_start]**学習成果の即時・多角的ポートフォリオ**: タスク完了の瞬間にスキルマップがアニメーションで成長し、新しいデジタルバッジが発行されます [cite: 9, 13, 16][cite_start]。この成果は外資就活ドットコムのプロフィールと連携できる将来性も示唆します [cite: 55, 59]。
* [cite_start]**ターゲットペルソナ**: 「30代後半、IT業界への転職を目指す非エンジニア営業職」を想定し、彼の「学習してもスキルを実感できず、転職でアピールできない」という”負”を解決するストーリーをデモで示します [cite: 13, 18]。

#### ✅ **技術・実装要件**

* [cite_start]**Webアプリ構成**: フロントエンドはNext.js (Vercel)、バックエンドはPython/FastAPI (Koyeb) の構成を取ります [cite: 1][cite_start]。これにより、運営から提供された環境を最大限に活用し、**「デモURL提出」**という最高評価の要件を迅速に満たします [cite: 5, 225]。
* [cite_start]**AIの役割分担**: ユーザーの課題分析（Claude）、具体的なタスク生成（GPT-5）など、複数のAIモデルを適材適所で活用し、「なぜこの技術を選んだか」という**技術的挑戦**の問いに明確に答えます [cite: 7, 13]。
* [cite_start]**データ連携**: Google Integrationsを活用し、Google Sheets経由で手動の学習ログなども取り込めるようにします [cite: 1]。

### 3. 開発・チーム運用戦略

「非エンジニア」である我々のチームが、最高のパフォーマンスを発揮するための運用ルールです。

#### ✅ **AI依存型開発スタイルとチーム演出**

* [cite_start]**基本戦略**: あなた（兄）がAIを駆使して意思決定と実装の大部分を担当する「偽エンジニア」戦略を貫きます [cite: 7, 219]。
* [cite_start]**「複数人協働」の証跡**: 妹アカウントで**最低3件以上のPull Request**（例：READMEの更新、UIの微修正）を作成・マージします [cite: 18, 220]。これにより、コミットが一人に偏らず、チームで開発したという客観的な証拠を残します。

#### ✅ **GitHub運用：「おもてなし」の精神で**

[cite_start]GitHubリポジトリは、審査員がプロダクトの裏側を見る「もう一つの顔」です。評価対象であることを強く意識し、徹底的に整備します [cite: 5, 172, 221]。

* [cite_start]**コミットログ**: 機能単位で小さくコミットし、メッセージは`feat: 〇〇機能を追加`のようにprefixルールを統一します [cite: 5, 18, 221, 225]。
* [cite_start]**PR運用**: `feature/*`ブランチからPull Requestを作成し、mainブランチにマージするフローを徹底します [cite: 5, 221, 225]。兄が妹のPRをレビューし、コメントを残すことで、協力の痕跡をさらに強化します。
* [cite_start]**README.mdの完成形**: これが最も重要です。以下の要素を網羅した「おもてなしのREADME」を作成します [cite: 5, 18, 177, 181, 221, 225]。
    * **デモURL**と**90秒のバックアップ動画**
    * すぐに使える**テスト用アカウント情報**
    * アーキテクチャ図と**技術選定理由**（なぜVercel/Koyeb/GPT-5なのか）
    * **技術的挑戦3点**（例：非認知能力の指標化、リアルタイム分析基盤、LLM連携の最適化）
    * **将来のKPI目標**（例：+2週で継続率45%達成）

### 4. プレゼンテーション戦略：7分間で勝利を確定させる

プレゼンは、我々の努力を結実させる最後の仕上げです。審査員の評価項目に沿って、一分の隙もないストーリーを構築します。

#### ✅ **デモは「二重化」で絶対に落とさない**

* [cite_start]**原則はURL提出**: 審査員が実際に触れる「動く実物」が最高評価です [cite: 5, 218, 225]。
* [cite_start]**保険は90秒動画**: ネットワーク障害やAPIエラーに備え、完璧な成功導線を収めた動画を必ず用意します [cite: 5, 14, 218, 222, 225]。スライドには「接続不良時は動画へ」と明記します。

#### ✅ **7分間スライド設計の型**

[cite_start]この構成と時間配分を厳守します [cite: 6, 222, 225]。

1.  **タイトル＆タグライン (0:30)**: 「あなたの学びを、資産に変える。AIパーソナル・ラーニング・レジャー」
2.  **“負”の解像度 (1:00)**: ペルソナ（30代営業職）の課題を、定量・定性データで提示。
3.  **解決コンセプト (1:00)**: 「非認知能力の可視化」など、差別化ポイントを3点に絞って提示。
4.  **デモ成功導線 (2:00)**: 「初回5タップ以内の成功体験」をURLと動画で見せる。
5.  **技術選定の妥当性＆挑戦3点 (1:00)**: READMEにも記載した内容を口頭で力強く説明。
6.  **将来性 (1:00)**: 2週/3か月/半年のロードマップとKPI目標を提示。
7.  **まとめ (0:30)**: 5つの評価項目に対応する我々の強みを、一言ずつ断定的に述べて締めくくる。

#### ✅ **リスク管理とその他**

* [cite_start]**SNS発信**: 予選前にハッシュタグ `#EGH` をつけて開発中のスクリーンショットを1回発信し、運営や審査員にチームを印象付けます [cite: 19, 225]。
* [cite_start]**協力アピール**: プレゼン中に「ここのUI改善は、チームメンバーの〇〇（妹）がこだわった点です」と一言入れることで、チーム協力をダメ押しでアピールします [cite: 19]。

このマスタープランに基づき、全ての思考と行動を一点に集中させることで、我々は必ずや勝利を掴むことができるでしょう。
